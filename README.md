# VITyarthi-Project-Java

# Project Title
EduGrievance - Grievance Redressal System

# Overview
EduGrievance is a desktop based Java application used to build to help students and teachers in managing grievances in schools, college and universities. It is easy to used and replaces traditional manual systems with unified, secure and transparent platform.

# Features
-> Secure user registration and login with role-based access control (Student/Admin).
-> Every grievance gets a unique ID and real-time status updates (Pending, In Progress, Resolved).
-> Students submit issues under categories like Academic, Hostel, or Infrastructure.
-> Admins can view, update, and resolve all submitted complaints through an easy table-based interface.
-> Intuitive Java Swing GUI that’s friendly for all users—no technical knowledge required.
-> All records saved safely in CSV format—no complex database setup needed.
-> User passwords are always protected using strong SHA-256 hashing.
-> Session management keeps your data accessible while you’re logged in.

# Technologies Used
-> Java SE 17
-> Java Swing
->Java IO (file storage)
-> Secure password handling with SHA-256

# Steps to install & run 
-> Install Java SE 17 or later on your computer.​
-> Download the EduGrievance source code.
-> Open the project in Java IDE or use command line.
-> Run the main class

# Testing the Application
-> All features were manually tested (Black Box method), focusing on what users would see and do.
-> Password hashing was checked by making sure the same input always produces the same secure output
-> Registered users and grievances are saved in plain text files; check users.txt or grievances.txt to verify data
-> Closing and reopening the app keeps your data safe
-> Session and role-based access features have been manually verified for correctness
